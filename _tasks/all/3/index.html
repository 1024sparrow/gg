# Механизм автопродолжений команд Разные команды мы раскидаем по разным скриптам. Эти скрипты располагаются в директории /usr/share/gg/commands Но должно быть место для общих подключаемых библиотек - /usr/share/gg/common У директории /usr/share/gg/commands должны быть иерархия скриптов - отдельно для каждой команды команд: ## Структура директории /usr/share/gg/commands Каждая команда представляется директорией. В этой директории * исполняемый файл main (без расширения, поскольку мы не привязываемся ни к какому интерпретатору). * директория arguments. Она используется исключительно (!!) для автопродолжения ввода аргументов команды. В этой директории располагаются - файлы. Имена которых задают дополнительные обязательные аргументы. Внутри этих файлов краткие описания, что они делают. - Исполняемые файлы. Выдают список дополнительных доступных параметров task утилита работы с задачами status remote clone commit push
